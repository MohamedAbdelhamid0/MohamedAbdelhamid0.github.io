RollJam Attack Explained

By Mohamed Abdelhamid

What is a RollJam Attack?

A RollJam attack is a method used to exploit vulnerabilities in keyless entry systems for vehicles. The attack involves capturing the radio frequency signals sent by a remote key fob when the owner attempts to lock or unlock their car.

Is This Attack Still Possible?

There are two primary variants of this attack, which differ based on vehicle age and security implementation.

Old Cars vs. Modern Cars

Older Vehicles (Approximately pre-2015-2020)

Older vehicles typically used static, unchanging codes for their keyless entry systems. For example, the unlock code might be fixed as "0010110" and the lock code as "0010101".

This implementation creates a significant security vulnerability. If an attacker captures these codes using a device like a Flipper Zero, they gain permanent access to the vehicle. The captured codes remain valid indefinitely, allowing unlimited unauthorized access.

This vulnerability prompted manufacturers to develop more secure solutions for modern vehicles.

Modern Vehicles

Modern vehicles employ rolling code technology to enhance security. This system uses an extensive sequence of codes shared between the key fob and vehicle receiver.

When a user presses the unlock button, the fob transmits a specific code (e.g., "0101101"). The vehicle validates this code, performs the requested action, and then removes that specific code from its accepted sequence. If the same code is transmitted again, the vehicle will reject it as invalid.

This security measure effectively prevents simple code capture attacks. However, as with any security system, determined attackers have identified potential vulnerabilities.

How Modern Cars Remain Vulnerable

Although the static code vulnerability has been addressed in modern vehicles, a sophisticated attack method known as RollJam can still compromise rolling code systems.

The RollJam Attack Vector

This attack exploits the synchronization mechanism between the key fob and vehicle:

  1. An attacker places a signal jammer near the vehicle
  2. When the owner attempts to unlock their car, the jammer blocks the signal from reaching the vehicle
  3. Simultaneously, the attacker captures the transmitted code
  4. The vehicle never receives the valid code, so it remains in the accepted code sequence
  5. The attacker now possesses a valid unlock code that the vehicle will accept

Can This Vulnerability Be Patched?

While no security system is completely invulnerable, manufacturers have implemented countermeasures to reduce this risk.

Code Synchronization Mechanics

Rolling code systems maintain synchronization through sequential code generation:

  • If the current valid code is "011010110", the next code will be "011010111"
  • Both the vehicle and key fob track this sequence

In a RollJam scenario:

  1. The vehicle and key fob are synchronized at code "0110"
  2. An attacker jams the signal when the user transmits code "0111"
  3. The vehicle never receives "0111", so it remains at code "0110"
  4. The key fob advances to code "1000"
  5. If the user presses the button again, transmitting "1000", the vehicle will synchronize to this new code
  6. All previous codes, including the captured "0111", become invalid

This synchronization mechanism limits the attacker to a single use of any captured code, significantly reducing the attack's effectiveness.
