RollJam Attack Explained

By Mohamed Abdelhamid

what is a RollJam attack?

this is a method used to exploit vulnerabilities in car that open remotely (not by inserting the key in the door). Basically the idea is capturing the codes sent by the remote when the unlock/lock button is clicked.

is that attack still possible?

there are two types of this attack

old cars vs modern cars

Old cars (~~ before 2015-2020 for some cars)

for old cars, they don't have changing code. the unlock code for a car is fixed for example to be 0010110 and the lock code is 0010101. this means that if a attacker used a gadget like a flipper zero captures the codes being sent by the keyfob to the car, he can later unlock the car whenever he wants (unlimited times since the code is fixed). obviously, this is very dangerous because the codes are fixed so once they're grabbed once, they can be used again and again. how was this solved in modern cars?

Modern cars

for modern cars, they now use rolling codes. what does that mean? basically the car has a very very large combination of codes (the same as the keyfob). when the user click the unlock button, that sends the code 0101101 as an example. at the same time, the car detects that this is a valid code so the car gets unlocked and removes this combination from its memory which means that if the car receives this code ever again, it won't be valid. this solved the way of capturing the code because now when the attacker captures the code, it won't be useful since the car already recognized it the first time and removed it from its memory. but as we know, nothing is 100% safe. there's still a vulnerability. What if a signal jammer was under the car?

how modern cars still got attacked

although the method used on the old cars can't be used on modern cars, there's still a vulnerability. if a signal jammer was under the car and the car owner pressed the unlock button, what will happen? the car won't receive the code but the attacker was sniffing for the code. this means that now the attacker got a single unlock code that is valid because the car didn't receive it and consequently didn't remove it from its memory.

can this method be patched?

well it's not 100% guaranteed but it can be. the codes on the keyfob are ordered meaning that if the current code is 011010110, the next code will be 011010111. here comes the confusing part, the car and keyfob are both on the same code 0110. an attacker adds a jammer near the car. the user pressed the unlock button that sends the code 0111 but the car didn't receive it. now the code the attacker has is 0111. the car is waiting at 0110 and the keyfob is at 1000. if the user presses any button again that sends 1000 to the car and the car reads it, the car will automatically sync to 1000 and mark all codes before 1000 as invalid. this means that the code the attacker has "0111" will be invalid.
